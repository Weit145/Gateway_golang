# Сервис-шлюз (GoLang)

## Обзор проекта

Этот проект функционирует как HTTP-шлюз, написанный на Go. Его основная роль заключается в предоставлении RESTful API, который проксирует запросы к gRPC-сервису `Auth_golang` (и, возможно, к другим внутренним gRPC-сервисам). Он преобразует HTTP-запросы в gRPC-вызовы и наоборот, предоставляя удобный интерфейс для функций аутентификации.

## Используемые технологии

*   **Go**: Основной язык программирования для приложения.
*   **HTTP**: Для обработки внешних клиентских запросов и предоставления RESTful API.
*   **gRPC Клиент**: Используется для связи с внутренними gRPC-сервисами, такими как сервис `Auth_golang`.
*   **cleanenv**: Используется для надежного управления конфигурацией, загружая настройки из YAML-файлов.
*   **slog**: Стандартная библиотека Go для структурированного логирования, обеспечивающая четкий и последовательный вывод логов.

## Структура проекта

Проект соответствует стандартной компоновке проекта Go:

*   `cmd/`: Содержит основную точку входа в приложение (`main.go`).
*   `config/`: Содержит файлы конфигурации для конкретных сред (например, `local.yaml`).
*   `internal/`: Содержит основную логику, включая:
    *   `config/`: Разбор конфигурации.
    *   `grpc/auth/`: Реализация gRPC-клиента для взаимодействия с сервисом `Auth_golang`.
    *   `http-server/handler/`: Содержит HTTP-обработчики, отвечающие за обработку входящих запросов и вызов соответствующих методов gRPC-клиента (например, `login`, `registration`, `confirm`, `refresh`, `logout`).
    *   `lib/`: Общие служебные библиотеки (например, `logger`, `response`).

## Сборка и запуск

### Предварительные требования
*   Установлен Go 1.25.5 или новее.

### Запуск приложения

Чтобы запустить приложение, выполните следующую команду из корневого каталога проекта:

```bash
go run cmd/main.go
```

Сервер шлюза запустится и будет прослушивать HTTP-соединения на настроенном порту, обычно `:8080`.

### Конфигурация

Настройки приложения управляются через YAML-файлы. По умолчанию используется `config/local.yaml`. Чтобы указать альтернативный файл конфигурации, установите переменную среды `CONFIG_PATH`:

```bash
CONFIG_PATH=/path/to/your/config.yaml go run cmd/main.go
```
